Actor

NxOgreActor.h

Actor is a dynamic (moving) RigidBody, that can collide with SceneGeometries and KinematicActors.

It is normal to inherit from the Actor to support visualisation in a RenderSystem, broadly
speaking any class that inherits the Actor class is called a Body.

Actors uses RigidBodyDescriptions to create it’s properities as well as a Matrix44 pose, a Shape
or group of Shapes. The RigidBodyDescription is required to have a mass is at least zero, or otherwise
a density of at least zero – but not both.

Example

RigidBodyDescription description;
description.mMass = 15.0f; // 15 kg

Actor* actor = mScene->createActor(shape, pose, description);

setCCDMotionThreshold

Sets the CCD Motion Threshold.

If CCD is globally enabled, it is still skipped for bodies
which have no point on any of their shapes moving more than CCDMotionThreshold distance in
one time step.

Hence, CCD is always performed if the threshold is 0.

The actor must be dynamic and the CCD motion threshold must be non-negative.

Arguments

Argument Description
Real thresh CCD Motion threshold

setMaxAngularVelocity

Lets you set the maximum angular velocity permitted for this actor.

Because for various internal computations, very quickly rotating actors introduce error
into the simulation, which leads to undesired results.

However, because some actors, such as car wheels, should be able to rotate quickly, you can override the default setting
on a per-actor basis with the below call. Note that objects such as wheels which are approximated with spherical or
other smooth collision primitives can be simulated with stability at a much higher angular velocity than, say, a box that
has corners.

Note — The angular velocity is clamped to the set value before the solver, which means that
the limit may still be momentarily exceeded.

Arguments

Argument Description
Real maxAngVel Max allowable angular velocity for actor

updateMassFromShapes

Recomputes a dynamic actor’s mass properties from its shapes

Given a constant density or total mass, the actors mass properties can be recomputed
using the shapes attached to the actor.

The mass of each shape is either the shape’s local density (default 1.0) multiplied by the shape’s volume
or a directly specified shape mass.

The inertia tensor, mass frame and center of mass will always be recomputed. If there are no
shapes in the actor, the mass will be totalMass, and the mass frame will be set to the center
of the actor.

If you supply a non-zero total mass, the actor’s mass and inertia will first be computed as
above and then scaled to fit this total mass.

If you supply a non-zero density, the actor’s mass and inertia will first be computed as above
and then scaled by this factor.

Note — Either totalMass or density must be non-zero.

Arguments

Argument Description
float density Density scale factor of the shapes belonging to the actor
float totalMass Total mass of the actor(or zero)

Return

bool — True if succesful
